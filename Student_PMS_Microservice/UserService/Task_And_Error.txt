Task And Error Notes

Priority 1 - High-Risk Runtime Errors
1. AuthController.Login does not await AuthService.Login.
   Current behavior returns Ok(Task<AuthResponseDTO>) instead of the actual login response.
   Impact: Login endpoint behavior is wrong at runtime.

2. JWT settings are read without null checks in startup and auth service code.
   Impact: Missing or malformed JWT configuration can cause null-reference failures.

Priority 2 - Unfinished Features
1. UserProfileController is empty.
2. UserProfileService methods return empty DTO/result objects and do not perform any real work.
3. UserProfileRepository methods return empty DTO/result objects and do not access storage.
4. UserProfile DTO classes are empty.
   Impact: The UserProfile feature is scaffolded but not implemented.

5. RabbitMQ integration files are fully commented out.
   Impact: Event-driven integration is unfinished.

6. UserMicroservice is only a placeholder with no outbound integration logic.
   Impact: Intended service-to-service communication is not implemented.

Priority 3 - Logic And Consistency Issues
1. RoleController injects validators for create and update but does not call them.
   Impact: Validation is inconsistent compared with UserController.

2. UserRepository throws plain Exception for not-found paths instead of using custom API exceptions.
   Impact: Error handling consistency is weak.

3. UserRepository.GetUserView may return null and then maps the result directly without explicit not-found handling.
   Impact: Null behavior depends on mapper behavior instead of explicit service logic.

4. AuthRepository constructor takes IConfiguration but does not use it.
   Impact: Unused dependency indicates incomplete or stale design.

Priority 4 - Quality Issues
1. There are many nullable warnings across DTOs, models, services, repositories, and Program.cs.
2. UserService contains a HomeController with a basic Add action that does not appear related to the service domain.
   Impact: This may be leftover/demo code rather than production API behavior.

Priority 5 - Tasks

2. Add strong JWT configuration validation during startup.
3. Implement or remove the unfinished UserProfile feature.
4. Re-enable and complete RabbitMQ only if messaging is required.
5. Apply validators in RoleController create and update paths.
6. Replace plain Exception usage with service-specific exceptions.
